  • UX Funnel

    Admin Flows
    Refers to admin-side groups that encompass large sections of the app. For example, the request flow starts at the student home page and ends at the public-facing request. Admin Flows and User Stories sometimes are the same, but with different focuses. Admin Flows tend to be more general than User Stories.
    User Stories
    Refers to an action a user might take in our app and the way the action is fulfilled. For example, “Guru checks ranking” is a user story.
    Scenes
    Refers to animated groups such as the projector. May be nested inside containers.
    Layouts
    Refers to the way that containers and components (including groups) are arranged on a given page. There are main layouts such as the ⅓ header and ⅓ sidebar as well as sublayouts like grid or tile layouts.
    Container Groups
    Refers to a group of containers that doesn’t take up an entire page but are larger and contain more detail than containers. An example of a container group would be the projector on the home page.
    Containers
    Refers to a group of components. Can also refer to layouts or large-format SVGs intended for animation.
    Component Groups
    Refers to multiple components grouped together that aren’t large enough or complex enough to be containers. For example, when you click “Logout” on the desktop navbar, there’s two buttons for confirming or canceling. Those two buttons together are a component group but not a container by itself. Other examples include the tile group on the dashboards, profile cards, or the request tile.
    Components
    Refers to elements made up of assets, text, and/or photos, such as buttons, progress bars, or user icons. Components are normally used as building blocks for component groups or containers.
    Assets
    Refers to photos and SVG icons used to make components. An asset cannot be used by itself in a container or component group.
